# 付费阅读服务（个人微信收款码）实施文档

> 场景：你一个人写文章 + 用微信付款码收款 + 单日收入 < 50 元。
> 不需要商户号、不需要营业执照、不需要回调，**整个支付流程完全自控**。

---

## 1. 整体流程

```
┌──────────┐     ┌────────────────────────┐     ┌──────────┐
│  读者端   │     │  Zeabur 单服务部署       │     │  作者端   │
│          │     │                        │     │          │
│ 1. 打开  │────▶│  Fastify API            │     │          │
│   文章   │◀────│  + Prisma + PostgreSQL │     │          │
│          │     │  + 静态前端             │     │          │
│ 2. 点   │     │                        │     │          │
│  "解锁" │────▶│ 3. 创建订单             │     │          │
│          │◀────│    返回你的收款码图片   │     │          │
│ 4. 微信  │     │                        │     │          │
│   扫码付款│    │                        │     │          │
│   备注   │     │                        │     │ 6. 微信  │
│   订单号 │     │                        │     │  收到通知│
│          │     │                        │     │          │
│ 5. 点   │     │                        │     │ 7. 后台  │
│  "我已  │────▶│ 8. 订单→AWAITING_CONFIRM│    │  点确认  │
│   支付" │     │                        │◀───│          │
│          │     │                        │────▶│          │
│ 9. 轮询 │     │ 10. 状态→CONFIRMED      │     │          │
│  命中 →  │◀────│     + 生成 unlock token│     │          │
│  看到完整│     │                        │     │          │
│   内容   │     │                        │     │          │
└──────────┘     └────────────────────────┘     └──────────┘
```

**核心**：用"作者手动确认"代替"微信支付 API 自动回调"。< 50 元/天的场景完全够用，反倒更可控。

**4 张表**：
- `PaymentQR` — 收款码图片（base64 存库）
- `Post` — 文章
- `Order` — 订单（PENDING / AWAITING_CONFIRM / CONFIRMED / REJECTED / EXPIRED）
- `Unlock` — 支付后生成，读者持 token 取完整内容

## 3. 关键流程详解

### 3.1 读者下单

```
1. 读者点 "立即解锁"
2. 前端 POST /api/orders { postSlug }
3. 后端：
   - 校验文章存在且已发布
   - 查最新一张 isActive=true 的收款码
   - 查是否已有该读者的 PENDING 订单（去重）
   - 生成 6 位订单号（手机里也好记）
   - 写 Order(status=PENDING, expiredAt=+30min)
   - 返回 { orderNo, amountCents, qrImage: <base64>, qrLabel, expiredAt }
4. 前端渲染：显示你的微信收款码图片 + 订单号 + 30 分钟倒计时
5. 提示读者："扫码付款时备注订单号"（便于你在微信通知里对照）
```

### 3.2 读者确认支付

```
6. 读者微信扫码 → 输入金额 → 备注订单号 → 付款
7. 你的微信收到通知（金额 + 备注订单号），你心里就有数了
8. 读者在前端点 "我已支付，等待作者确认"
9. 前端 POST /api/orders/:orderNo/paid
10. 后端：Order.status = AWAITING_CONFIRM
11. 前端开始每 3 秒轮询 GET /api/orders/:orderNo
```

### 3.3 作者确认

```
12. 你登录 #/admin/orders → "待我确认" 列表里看到该订单
13. 你掏出手机看一眼微信通知：
    - 有这笔钱 → 点 [确认收款]
    - 没收到 → 点 [拒绝]（可填原因）
14. 点 [确认收款] → POST /api/admin/orders/:id/confirm
15. 后端事务：
    - Order.status = CONFIRMED
    - 生成 Unlock.token
16. 读者页面轮询命中（status=CONFIRMED + unlockToken）：
    - 写入 localStorage
    - 重新加载文章 → 展示完整内容
    - 之后访问该文章时，X-Unlock-Token header 自动带 → 直接看完整
```

### 3.4 防白嫖机制

- **每篇文章每个读者最多 1 个 PENDING/AWAITING 订单**：通过 `reader_token` cookie 的 sha256 指纹去重
- **订单 30 分钟过期**：过期后状态自动 EXPIRED，读者需刷新重新创建
- **`content` 字段绝不返回给未解锁用户**：公开 API 只返 `preview`，完整内容必须带 `X-Unlock-Token` header
- **作者可在 24 小时内拒绝**：发现漏确认或多收都可撤销
- **unlockToken 是 32 字节随机 hex**：暴力破解不现实

```bash
# server/.env
NODE_ENV=development
PORT=3000

ADMIN_USERNAME=admin
ADMIN_PASSWORD=你的强密码_至少8位
ADMIN_DISPLAY_NAME=你的笔名

# 32+ 字符随机串：openssl rand -hex 32
ADMIN_JWT_SECRET=xxxxxxxx
READER_TOKEN_SECRET=yyyyyyyy
```

**没有** WECHAT_APPID / WECHAT_MCHID / API key / 证书，整个 `.env` 干净到不行。

## 5. 本地开发

```bash
# 1. 启动 PostgreSQL
docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=postgres --name pg-paid postgres:16

# 2. 初始化项目
cd server
cp .env.example .env
# 编辑 .env：填 ADMIN_PASSWORD / 随机密钥
npm install
npm run db:migrate -- --name init
npm run db:seed        # 创建占位收款码 + 示例文章

# 3. 启动
npm run dev
# → http://localhost:3000
```

**首次使用**：
1. 打开 `http://localhost:3000/#/admin` → 输入 `ADMIN_PASSWORD` 登录
2. 跳到 `#/admin/qrs` → **上传你的真实微信收款码**（占位 1x1 图要先换掉）
3. `#/admin/posts` → 改示例文章或新建

## 6. Zeabur 部署

### 6.1 推代码

```bash
cd "/Users/huojin/Desktop/有偿文章阅读"
git init
git add .
git commit -m "init: 付费阅读 MVP"
git remote add origin git@github.com:你的用户名/你的仓库.git
git push -u origin main
```

### 6.2 创建服务

1. Zeabur 控制台 → New Service → 选这个仓库
2. **Root Directory 填 `server`**（仓库根是有偿文章阅读/，要选子目录）

### 6.3 添加 PostgreSQL

1. Zeabur 控制台 → New Service → Marketplace → **PostgreSQL**
2. 创建后自动注入 `DATABASE_URL` 到主服务

### 6.4 配置环境变量

把第 4 节的所有变量填到主服务 Environment Variables。`ADMIN_PASSWORD` 用你常用的强密码。

### 6.5 启动命令 + 首次 migration

Zeabur 默认启动命令是 `npm start`，但**首次部署需要先跑 migration**。

在 Zeabur 的「Start Command」字段改成：

```bash
npx prisma migrate deploy && npm start
```

### 6.6 绑定域名

Settings → Domains → Generate Domain（得到 `xxx.zeabur.app`）或绑自定义域名。

### 6.7 验证

```bash
curl https://你的域名/health
# → {"ok":true,"ts":...}
```

打开 `https://你的域名/` 应该看到首页。

## 7. 收款码上传流程

**首次部署完成后，必须做这一件事**：

1. 打开你的微信 → 我 → 服务 → 收付款 → 收款码 → 保存图片
2. 浏览器打开 `https://你的域名/#/admin` → 登录
3. 跳到 `/#/admin/qrs` → 选择图片 + 填备注（"微信-主号"）→ 上传
4. 上传后该码自动启用，旧码停用（**始终只有 1 张激活**）
5. 现在读者下单就会看到这张码

**图片要求**：
- PNG / JPG
- ≤ 200KB（建议先压缩，微信收款码通常 < 50KB）
- 建议用微信原图直接保存，无需美化

## 8. 日常运营工作流

### 8.1 发布新文章

`/#/admin/posts/new`：
- Slug 写英文短横线，如 `my-thoughts-2024-12`
- 公开预览写 20-30% 摘要（这是读者判断值不值的唯一依据）
- 完整内容写付费后才能看的部分
- 价格填分（如 990 = ¥9.9）
- 状态选 PUBLISHED

### 8.2 处理订单

`/#/admin/orders` 默认显示"待我确认"列表。每条订单显示：
- 6 位订单号
- 文章标题
- 金额 + 创建时间

操作流程：
1. 读者付款 → 你微信收到通知（备注里有订单号）
2. 登录后台 → 该订单号出现在"待我确认"
3. 对照微信通知：
   - 钱到了 → [确认收款]（系统自动给读者解锁）
   - 没到 → [拒绝]（填原因，读者端会看到提示）

### 8.3 修改价格

`/#/admin/posts` → 找到文章 → 编辑 → 改价格 → 保存。已存在的 unlock 不受影响（已买的就是已买的）。

## 9. 数据安全与备份

| 项目 | 方案 |
|------|------|
| 数据库备份 | Zeabur PostgreSQL 自动每日备份，Hobby 计划保留 7 天 |
| 文章内容 | 存数据库，建议本地也保存一份原始 markdown |
| 收款码图片 | 建议本地保留原图，重新上传即可恢复 |
| 强密码 | `ADMIN_PASSWORD` 至少 12 位 + 大小写 + 数字 + 符号 |
| JWT 密钥 | 32 字节随机串（`openssl rand -hex 32`），丢了所有人需重新登录 |
